LumenKB Project Review

把碎片信息变成可复习、可调用的个人知识库

LumenKB 是一个面向学习者、知识工作者和求职作品集场景的个人知识管理应用。它把「记录内容」「AI 提炼」「个人感悟」「复习巩固」放在同一条闭环里，强调知识不是收藏，而是能被再次理解和使用。

项目定位

LumenKB 不是单纯的笔记列表，也不是只做聊天问答的 AI 工具。它定位为一个轻量的个人知识库，让用户把读到的文章、技术资料、复盘材料和灵感记录为结构化文档，再通过 AI 摘要与复习机制持续加工。

项目名称中的 Lumen 表达「照亮」和「理解」，KB 代表 Knowledge Base。产品重点是把用户的输入从信息层推进到知识层。

目标用户

  • 正在学习新领域的人，需要把资料转化为自己的理解。
  • 知识工作者，需要在项目、阅读、研究和输出之间复用知识。
  • 求职者或作品集展示者，需要展示自己对 AI 产品、前后端和数据持久化的完整实现能力。
  • 内容创作者，需要把素材、摘要、标签和复盘结果组织起来。

核心痛点

  • 收藏很多资料，但缺少主动加工，之后很难再次使用。
  • 笔记内容越积越多，检索和分类成本升高。
  • 读完内容后没有即时摘要，后续复盘需要重新阅读全文。
  • 学习记录和复习流程割裂，无法判断哪些知识已经掌握。

核心流程

1

输入内容

用户新建知识记录，补充标题、分类、标签和正文。

2

AI 提炼

通过 Gemini 生成短摘要，帮助用户快速抓住关键观点。

3

沉淀理解

在阅读页写下个人感悟，将外部信息转化为自己的判断。

4

复习巩固

每日复习队列根据掌握状态推进，形成持续回顾闭环。

当前功能

  • 知识文档的创建、展示、分类和标签管理。
  • 知识库列表页，支持搜索、分类筛选和掌握状态筛选。
  • 文档详情页，展示正文、预计阅读时间、AI 摘要和个人感悟。
  • AI 摘要接口，后端调用 Google Gemini 生成结构化摘要。
  • 每日复习页，使用翻卡交互记录「记得、模糊、忘记」等结果。
  • 个人资料读取与更新，以及系统内置示例文档同步。

技术栈

React TypeScript Vite React Router Tailwind CSS Express PostgreSQL Google Gemini API Vercel

架构说明

前端负责知识库、编辑、阅读和复习体验；后端提供文档、用户资料、个人感悟和 AI 摘要 API；数据库负责持久化文档、摘要、标签、复习状态与感悟内容。

项目亮点

  • 完整闭环：从输入、摘要、阅读、感悟到复习，不停留在静态笔记。
  • AI 用得克制：AI 负责提炼摘要，最终理解仍由用户通过感悟和复习完成。
  • 数据模型清晰：文档、摘要、标签、复习状态、感悟内容分工明确。
  • 展示友好：同时提供静态 HTML 和 Markdown 两个入口，方便 AI、面试官和部署平台直接读取。

下一版迭代

  • 增加语义搜索和相似文档推荐，让知识之间形成更强连接。
  • 引入间隔重复算法，根据复习结果动态安排下一次复习时间。
  • 支持 Markdown 导入导出，方便用户迁移和备份个人知识库。
  • 增加知识图谱视图，展示标签、分类和文档之间的关系。
  • 完善鉴权与多用户隔离，让项目从作品集演示走向真实使用。

面试表达

我做 LumenKB 的出发点是解决个人知识管理里「只收藏、不消化」的问题。项目把知识记录、AI 摘要、个人感悟和每日复习串成一个闭环。前端用 React 和 Vite 实现多页面体验，后端用 Express 提供文档、用户资料、感悟和 AI 摘要接口，数据持久化在 PostgreSQL 中。这个项目重点展示的是我把 AI 能力嵌入具体业务流程的能力，而不是为了 AI 而 AI。
